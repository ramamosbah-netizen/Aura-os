## Part 1 — What was built & verified (Phase 0a)

A runnable, clean foundation. No modules yet — kernel spine only.

**Monorepo** (`pnpm` + Turborepo):
```
aura-os/
├ apps/api        @aura/api      NestJS host (boots on :4000/api)
├ core            @aura/core     KERNEL — event store + bus + tenant context
├ shared          @aura/shared   framework-free types: Money, Id, DomainEvent, event catalog
├ modules/ intelligence/ infrastructure/   (placeholders + READMEs)
└ docs/           the blueprints + this report
```

**Implemented & proven:**
- Append-only **event spine** end-to-end: `DomainEvent → EventStore → EventBus → subscriber`.
- `pnpm install` (303 pkgs) → `pnpm build` (turbo orders shared → core → api) → **all green**.
- Live verification: `GET /api/health` → `{status:ok}`; `POST /api/events` (a `procurement.po.approved`) returned a stamped event; the bus subscriber logged `▶ procurement.po.approved (PurchaseOrder:po-1001) tenant=dev-tenant`.
- Committed (`91adacd`, 40 files).

**Deliberate stand-ins (replaced in Phase 0b):** in-memory `EventStore` (→ Postgres + outbox); singleton `TenantContext` (→ AsyncLocalStorage); `dev-tenant` default (→ real auth).

**Risk posture:** ~10% of features, but ~40% of the *technical* risk retired — the hard part (getting the platform kernel right once) is underway. This is the correct order.

## Part 2 — Architecture v2: the refinements (amends the blueprint)

Driven by the strategic review. **No module is built until the kernel below is complete.**

### 2.1 Kernel (T0) — expanded scope (the real "OS")
| Kernel service | Why it's kernel (used by *every* module) |
|---|---|
| **Identity & Access Platform** | Full hierarchy `Tenant → Company → Business Unit → Department → Team`, plus `Role → Permission → Scope` (RBAC **+ ABAC**). Enables `User → Company A → Project X → Procurement`. Foundation of everything. |
| **Event store + outbox + bus** | done (0a) → Postgres-backed in 0b |
| **Platform Workflow Engine** | generic, data-driven, **any** module registers definitions — not PO-specific |
| **DMS Platform** | priority #2 after events. Tender/Drawing/Submittal/Contract/Invoice are all Documents → build the substrate (storage · versioning · metadata · OCR hook · search) now |
| **AI Provider Layer** | **moved from Intelligence → Kernel.** Claude/OpenAI/Gemini/Azure used system-wide; the `ai-provider` reference written earlier lands in `core` |
| **Integration Platform** | from day one, even without connectors: Webhook · REST gateway · Import · Export · Queue |
| Numbering · Audit · Notifications | platform services |

### 2.2 Module map changes
- **+ Contract Management** (new standalone module). The deal chain is **`CRM → Tender → Contract → Project`** — the contract is the pivot. Contains: Client Contracts · Subcontracts · Obligations · Milestones · Bonds · Guarantees · Insurance · Claims · Correspondence · Renewals · Contract Risks.
- **Project Controls split from Projects** (mirrors Primavera P6 / EcoSys / Oracle Unifier): **Projects** = execution; **Project Controls** = CBS · Cost · EVM · Forecast · Cashflow.
- Net business modules: **18** (was 16) — Contract Management added, Projects → Projects + Project Controls.

### 2.3 Experience — Workspace-first
The home is **Workspace ("My Work")**, not a Dashboard. It aggregates the user's actionable items across all modules:
```
My Work →  5 Approvals · 2 RFQ · 1 Contract · 3 Tickets · 4 NCR  → click straight in
```
Traditional module menus exist but are secondary. This personal-work surface (AURA's "Today Brain") is the platform's signature.
