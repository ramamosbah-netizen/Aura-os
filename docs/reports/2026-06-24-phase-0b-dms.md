# Report — Phase 0b.4a: Document Management substrate (kernel DMS)

**Date:** 2026-06-24 · **Repo:** `Desktop/aura-os` (local, branch `main`) · **Increment:** Phase 0b step 4, part 1 of 3 (DMS · Workflow · Integration).

> Priority #2 after the event store: Tender/Drawing/Submittal/Contract/Invoice are all **Documents**. Build the substrate now or rewrite half the system later.

---

## What was built

**Framework-free model in `@aura/shared`** (`src/dms/document.ts`):
- `Document` (linked to any aggregate via `aggregateType`/`aggregateId`, like `DomainEvent`) + `DocumentVersion` (append-only history) + `DocumentStatus`.
- Pure helpers: `makeDocument` (starts at v1), `makeDocumentVersion`, `nextVersionNumber`, `storageKeyFor` (deterministic, sanitized `tenant/aggregate/doc/vN-file`), and `DMS_EVENT` constants.

**Ports + impls + service in `@aura/core`** (`src/dms/`), wired into `CoreModule`:
- `DocumentStorage` port (binary backend) + `LocalDocumentStorage` (filesystem, sha256 checksums, boot-safe default). Supabase Storage / S3 plug in behind the same port.
- `DocumentStore` port (metadata) + `PostgresDocumentStore` (atomic doc+version transaction) + `InMemoryDocumentStore` fallback — selected from `DATABASE_URL` like the event store.
- `DmsService` — the facade modules call: stores bytes → persists metadata → **emits `dms.document.*` on the event spine**. `createDocument`, `addVersion`, `get`, `list`, `readContent`.

**Migration** `0002_kernel_documents.sql` — `aura_documents` + `aura_document_versions` (FK, `unique(document_id, version)`, indexes, RLS-locked). **Demo controller** `POST/GET /api/documents` (+ `/:id/versions`).

## Verified live (dedicated Supabase project)

`pnpm db:migrate` applied `0002` (idempotently skipped `0001`). Booted, then over HTTP:
- `POST /api/documents` (a contract) → persisted; v1 stored (checksum `5ad77801…`, 16 bytes, key `dev-tenant/contract/contract-9/<id>/v1-Main_Contract.txt`).
- `POST /api/documents/:id/versions` → `currentVersion=2`, **2 versions** read back from Postgres; list-by-kind returns 1.
- Bytes on disk: `v1-Main_Contract.txt` + `v2-contract.txt` under `.aura-storage` (gitignored).
- **Spine integration proven:** `[Dms] Created …` → `[Events] ▶ dms.document.created` → `OutboxRelay Relayed 1 event(s)`, then the same chain for `dms.document.version_added`. DMS mutations ride the transactional outbox from 0b.2.

`pnpm build` → 3/3 · `pnpm test` → **27/27** (12 identity + 9 AI + **6 new DMS**).

## Decisions

- **Documents link to any aggregate** — same `aggregateType`/`aggregateId` shape as events; one substrate, every module attaches to it.
- **Append-only version history** — `currentVersion` on the doc, immutable rows in `aura_document_versions`.
- **Storage is a seam** — local FS ships now; Supabase Storage / S3 are drop-in adapters (the `SUPABASE_URL`/service key are already in `.env.local` for that next step).
- **Every mutation emits on the spine** — so projectors / Intelligence see document changes without DMS knowing about them.
- **Known follow-up (atomicity):** `DmsService` persists metadata (atomic doc+version) then appends the event in a separate transaction. Folding the event into the document transaction via `PostgresEventStore.appendWithClient` is the strict-atomic-outbox upgrade — the machinery exists; deferred to keep this slice contained.
