# Report — T1.4: Projects module (Projects v1) — the final deal-chain link

**Date:** 2026-06-24 · **Repo:** `Desktop/aura-os` (local, branch `main`) · **Increment:** T1 step 4 — **Project**, the last link (CRM → Tender → Contract → **Project**). This **completes the deal chain** end to end.

> A Project is the delivery of a signed (active) contract. With it, the platform models the whole lifecycle — lead → bid → contract → delivery — every hop a snapshot reference, no cross-module join anywhere.

---

## What was built

**`modules/projects` — the `@aura/projects` package:** framework-free `Project` domain holding `contractId`+`contractTitle` **and** `accountId`+`accountName`; store port + Postgres/in-memory (filters tenant/status/account/**contract**); `ProjectService` (owns data, access seam `assert('projects.project.create')`, emits `projects.project.created` with both refs in the payload). 4 vitest tests.

**API** (`apps/api`): `ProjectsController` (`POST/GET /api/projects/projects`, `?contractId` filter). Wired into `AppModule`. Migration `0008_projects_projects.sql` (`aura_projects_projects`, indexed tenant/status/account/contract, RLS).

**Web** (`apps/web`): **Projects** added to the Deal-chain nav → `/projects/projects` page + BFF route + `components/project-create.tsx`. The page fetches projects **and active contracts in parallel** (`/api/contracts/contracts?status=active`); the form's dropdown is those active contracts, and picking one inherits its account + value — the chain arriving at delivery, composed through the contract.

## Decisions

- **Chain complete, still join-free.** Four modules, four tables (`aura_crm_*`, `aura_tendering_*`, `aura_contracts_*`, `aura_projects_*`), and the entire lead-to-delivery lifecycle is traceable purely via snapshot references + event payloads. A consumer of the spine can reconstruct the whole chain from the events alone.
- **The template is now boring (the goal).** Fourth clone, same shape, near-zero surprises — adding a bounded context is a known, low-risk, ~one-pass operation on this kernel.
- **Access seam wired, enforcement awaits auth** — `actorId = null` in dev passes through, shared by all four modules. The one standing structural gap.
