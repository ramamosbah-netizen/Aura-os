# Report ‚Äî Auth JWKS: hosted-IdP (Supabase) token verification

**Date:** 2026-06-25 ¬∑ **Repo:** `Desktop/aura-os` (local, branch `main`) ¬∑ **Increment:** finishes the auth story ‚Äî the API now verifies **asymmetric IdP tokens** (Supabase/Auth0/‚Ä¶) alongside the HS256 dev tokens.

## What was built

**`@aura/shared` ‚Äî `verifyJwtWithJwks(token, jwks)`** (`src/identity/jwks.ts`): a dependency-free JWT verifier over a JWKS, using only `node:crypto` (`createPublicKey({format:'jwk'})` + `crypto.verify`). Supports **RS256/384/512, ES256/384/512** (ECDSA needs `dsaEncoding:'ieee-p1363'` ‚Äî JWT sigs are raw R‚ÄñS, not DER) and **EdDSA**. Selects the key by `kid`, enforces the `alg`, checks `exp`. **6 vitest tests** (valid RS256, valid ES256, tampered payload, wrong-key/same-kid, expired, alg=none/malformed) ‚Äî all generate real keypairs and mint/verify.

**`@aura/core` ‚Äî `AuthService`** now tries two verifiers in order:
1. **Hosted-IdP JWKS** when `AUTH_JWKS_URL` (or `SUPABASE_JWKS_URL`) is set ‚Äî a `JwksCache` fetches + caches the keys (10-min TTL, refetch-once on a key-miss for rotation), and `toContext` maps Supabase-style claims (`sub` ‚Üí actor; tenant from `tenantId` / `app_metadata.tenant_id` / `AUTH_DEFAULT_TENANT`).
2. **Self-issued HS256** when `AUTH_JWT_SECRET` is set (the existing dev path).
`enabled` is true if *either* is configured; `canMint` (HS256 only) now gates the dev-login/dev-token endpoints. `contextFromHeader` became async; the bootstrap middleware awaits it.

## Decisions

- **Provider-agnostic, dependency-free** ‚Äî nothing is Supabase-specific; any JWKS-publishing IdP works. No new packages.
- **Supabase is now config-only** ‚Äî set `SUPABASE_JWKS_URL` (the project's `‚Ä¶/auth/v1/.well-known/jwks.json`, already provided) and `AUTH_REQUIRED`; real RS256/ES256 Supabase tokens verify via the exact path proven above. Tenant comes from `app_metadata.tenant_id` (set it on the Supabase user) or the default.
- **Same staged seam** ‚Äî no verifier configured ‚áí auth off ‚áí dev pass-through; the API always boots.
