## 1. Database Schema
Created PostgreSQL tables via migration `0026_fleet.sql` under namespaces `aura_fleet_*` to store vehicle profiles, fuel entries, and maintenance logs:

```sql
-- 1. Fleet Vehicles
CREATE TABLE IF NOT EXISTS public.aura_fleet_vehicles (
  id                    UUID PRIMARY KEY,
  tenant_id             TEXT NOT NULL,
  company_id            TEXT,
  make                  TEXT NOT NULL,
  model                 TEXT NOT NULL,
  year                  INTEGER NOT NULL,
  plate_number          TEXT NOT NULL UNIQUE,
  registration_expiry   DATE,
  status                TEXT NOT NULL DEFAULT 'active',
  driver_employee_id    UUID REFERENCES public.aura_hr_employees(id) ON DELETE SET NULL,
  created_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 2. Fuel Log Registry
CREATE TABLE IF NOT EXISTS public.aura_fleet_fuel_logs (
  id            UUID PRIMARY KEY,
  tenant_id     TEXT NOT NULL,
  company_id    TEXT,
  vehicle_id    UUID NOT NULL REFERENCES public.aura_fleet_vehicles(id) ON DELETE CASCADE,
  date          DATE NOT NULL,
  liters        NUMERIC(10,2) NOT NULL,
  cost          NUMERIC(10,2) NOT NULL,
  odometer      NUMERIC(15,2) NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 3. Preventative Maintenance Records
CREATE TABLE IF NOT EXISTS public.aura_fleet_maintenance (
  id            UUID PRIMARY KEY,
  tenant_id     TEXT NOT NULL,
  company_id    TEXT,
  vehicle_id    UUID NOT NULL REFERENCES public.aura_fleet_vehicles(id) ON DELETE CASCADE,
  date          DATE NOT NULL,
  description   TEXT NOT NULL,
  cost          NUMERIC(15,2) NOT NULL DEFAULT 0.00,
  status        TEXT NOT NULL DEFAULT 'scheduled',
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## 2. API Integration
- **NestJS Service & Module:** `FleetService` orchestrates CRUD methods, asserts permissions (`fleet.vehicle.create`, `fleet.fuel.create`, `fleet.maintenance.create`, etc.), appends auditable log events to the global event store, and integrates database transaction runners (`TxRunner`). The module automatically injects the `Postgres` adapter when a database connection pool is present, or falls back to the unit-testing `InMemory` adapter.
- **NestJS Controller:** Exposes REST routes under prefix `/api/fleet` (`/vehicles`, `/fuel`, `/maintenance`) to manage vehicle registries, log refuelings, and resolve scheduled or completed service items.
- **Next.js BFF Integration:** Implemented Dynamic Next.js App Router API handlers forwarding request actions with tenant context and authorization:
  - `GET/POST` `/api/fleet/vehicles`
  - `DELETE` `/api/fleet/vehicles/[id]`
  - `GET/POST` `/api/fleet/fuel`
  - `GET/POST` `/api/fleet/maintenance`
  - `PUT` `/api/fleet/maintenance/[id]/complete`

## 3. UI Component Details
- **Vehicles Registry:** Supports asset registration (make, model, plate, manufacture year, registration expiry monitoring), and assigns active staff from the HR employees database directory as drivers.
- **Fuel Logs:** Tracks liters filled, total cost in AED, and odometer readings to trace fuel efficiency across assets.
- **Preventative Maintenance:** Supports scheduling maintenance items (e.g. oil change, brake replacements), showing estimated costs, and resolving scheduled events via inline prompt forms to log actual costs.
