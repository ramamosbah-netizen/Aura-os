## 1. #21 — Published SDK (`@aura/sdk`)

- New workspace package `packages/sdk`: hand-written core (`client.ts`) + generated surface
  (`generated.ts`, **646 operations** from the live OpenAPI doc at `/api/docs-json`).
- The core carries the platform contracts: **`AuraApiError`** maps the enforced error
  taxonomy (400→VALIDATION, 401→AUTH, 403→FORBIDDEN, 404→NOT_FOUND, 409→CONFLICT,
  429→RATE_LIMITED, 5xx→SERVER), `Idempotency-Key` per call, `Page<T>`, `setToken` rotation.
- Generator: `scripts/generate-sdk.mjs` — uses `SPEC_URL` or boots the built API in-memory
  (no DB) on a scratch port. **CI drift gate**: the `deploy-readiness` job regenerates
  against the live booted API and `git diff --exit-code`s the generated file.
- Verified live: boot → `authLogin` → `crmAccountsCreate` (with idempotency key) →
  `crmAccountsPaged` → missing id rejects as `AuraApiError NOT_FOUND`. 5 unit tests.
- Honest limits: payload types are `unknown` until DTOs gain swagger schemas; the old
  `SdkGeneratorService` command-stub emitter in core is superseded.

  **numbering gaps**, @DOWN presence. Verified on the current 136-file chain.

  cross-context references (tenders→accounts … GRNs→POs), tenant-scoped `not exists`

  rows only** — pending outbox never moves) + `aura_audit_log` older than N months (default
  12) into `*_archive` twins, transactional batches; **dry-run by default**; CI smoke.
- Policy + cadences + run log: `docs/runbooks/data-lifecycle.md`.
