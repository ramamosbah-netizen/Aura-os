# Account 360 — the Customer Command Center

**Date:** 2026-07-11 · **Branch:** `feat/account-360`
**Why:** the account detail page was a near-empty record (status, industry, website, three
static links) — nothing like the AURA OS vision where the **Account is the persistent
commercial party at the head of the deal chain** and opportunities/tenders/quotations/
contracts/projects are the transactions that flow through it. One client ⇒ many deals.

---

## 1. What shipped

### Domain (migration **0144**)
`Account` grew the commercial-profile fields the page shows: `phone`, `email`,
`billingAddress`, `source`, `paymentTerms` — create/update DTOs and the PG store carry
them. `OpportunityFilter` gained `accountId` (pg + in-memory) so the chain query pushes down.

### API — `GET /crm/accounts/:id/summary` (`Account360Controller`)
One fetch composes the whole center (all by `accountId`; invoices match the customer-name
snapshot since finance references by name):
- **contacts · opportunities · tenders · quotations · contracts · projects · activities**
- **receivables**: invoiced / paid / outstanding / overdue (due date vs today)
- **commercial summary**: pipeline value (open opps), active opportunities, open/total
  tenders, quotation/contract/project counts, won value (Σ contract values), outstanding AR,
  and a derived **health** flag (`watch` when AR is overdue, `good` with live business, `new`)
- **timeline**: synthesized from the records themselves (account created, contact added,
  opportunity, tender registered, quotation issued, contract awarded, project started,
  invoices) — newest first, capped at 60, each entry linking to its record.

### Web — `/crm/accounts/[id]` rebuilt as the 360
- **Header**: name · status · **health pill** · industry/owner/client-since/source ·
  quick actions (+ Quotation, + Tender, + Contact)
- **Commercial summary** stat bar (overdue AR renders red)
- **Deal-chain strip**: `◆ Account → Opportunities n → Tenders n → Quotations n →
  Contracts n → Projects n` — populated stages highlighted, each linking to its module
- **Tabs**: Overview (full profile grid incl. main contact fallback for phone/email) ·
  Contacts · Opportunities · Tenders · Quotations · Contracts · Projects · Activities ·
  Financials (AR mini-dashboard + payment terms) · Timeline — every tab lists the ACTUAL
  records with status pills, values, and Open→ links.

## 2. Verification (live, dev DB)
`GET /crm/accounts/:id/summary` on the seeded "Acme" account: receivables matched by name
(1,000 outstanding, overdue ⇒ health `watch`), profile PATCH with the five new fields
persisted, contact + tender created and appeared in the chain strip (`Tenders 1`
highlighted), tab counts, overview grid, and the timeline (tender → contact → account →
invoice, newest first) all verified in the browser. crm tests 16 · api/web builds green ·
SDK regenerated (**670 ops**) · migration gate green (144, @DOWN).

## 3. Notes & follow-ons
- Customer invoices reference customers by NAME snapshot — a future `accountId` column on
  `aura_finance_customer_invoices` would make Financials exact under renames (S).
- Documents tab deferred: DMS attachments hang off individual records today; an
  account-level roll-up needs a DMS query seam (register #24 territory).
- Demo-seed opportunities use non-uuid account ids (`a-1`) — the seeder should reference
  the seeded uuid accounts so demo accounts show a fuller chain (S).
- Timeline is synthesized from record snapshots; wiring it to `aura_events` would add
  status-transition granularity (sent→accepted, won/lost) later.
